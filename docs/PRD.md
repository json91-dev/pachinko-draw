# 개요
닉네임을 입력한 플레이어들이 각자의 색상 구슬을 파칭코 방식으로 떨어뜨려, 바닥의 구멍 1개에 가장 많이 넣은 플레이어가 당첨되는 웹 게임이다. 기술 스택은 Next.js와 Matter.js(물리 엔진)를 사용한다.

---

# 화면 구성

## 설정 화면
전체화면 입력 UI로 구성된다. 화면 하단에 "이름들을 입력하세요" 제목과 큰 textarea, 맵 선택 드롭다운, "시작" 버튼이 위치한다. 콤마(,)로 구분하여 닉네임을 입력하면 **화면 우측 상단**에 플레이어 목록이 실시간으로 자동 생성·제거된다.

**플레이어 목록 (우측 상단 고정, 설정 화면 & 게임 화면 공통)**
- 카드 형태가 아닌 간단한 텍스트 목록으로 표시
- 우측 정렬, 세로로 일렬 나열
- 각 행 형식: `[구슬 수]개 [닉네임] #[순서]`
  - 예) `10개 닉네임1 #1` / `33개 닉네임2 #2`
- 텍스트 색상은 해당 플레이어의 구슬 색상과 동일
- 게임 중에는 구슬 수가 실시간으로 점수(구멍에 넣은 개수)로 업데이트됨
- 이 목록이 설정 화면의 플레이어 미리보기이자 게임 중 HUD 역할을 동시에 수행

공백은 자동으로 제거하지 않으며 입력값 그대로 유지한다. 최소 2명에서 최대 30명까지 참여할 수 있다. 빈 닉네임과 중복 닉네임은 허용하지 않으며, 유효성 검사를 통과해야 "시작" 버튼이 활성화된다.

**맵 선택 드롭다운:** 닉네임 입력 영역 내에 `<select>` 드롭다운으로 배치한다.
- **기본맵:** 핀이 지그재그 격자로 배치된 기본 파칭코 보드
- **빙글빙글 물레방아:** 물레방아가 있는 영역에만 핀이 없고 나머지는 기본맵과 동일한 핀 배치

게임 시작 버튼을 누르면 입력창(textarea, 드롭다운, 버튼) 영역은 즉시 사라지고 게임 Canvas로 전환된다. 우측 상단 플레이어 목록은 그대로 유지된다.

## 게임 화면
Canvas 기반으로 렌더링되는 파칭코 보드는 핀(페그)과 좌우 벽, 상단 경계로 구성된다. 전체 배경은 검정색(#000000)이다.

**캔버스 해상도:** 고정 가상 해상도(1600×900)를 기준으로 핀·물리 좌표를 계산하고, CSS transform scale로 뷰포트에 맞게 스케일링한다. 이를 통해 브라우저 크기와 무관하게 동일한 레이아웃을 유지한다.

**대포:** 보드 상단 중앙에 `docs/images/cannon.png` 이미지로 대포를 렌더링하며, 구슬은 이 대포에서 발사된다. 대포는 좌우로 천천히 스윙(회전)하며, 현재 가리키는 각도 방향으로 구슬을 연속 발사한다.

**핀 배치:** `docs/images/pin_128.png` 이미지를 사용하며, 지그재그 격자 형태로 일정한 간격에 배치한다. 핀 위치는 캔버스 가상 해상도 기준 비율로 계산한다.
- 홀수 행(1·3·5·…행): 핀 6개
- 짝수 행(2·4·6·…행): 핀 7개
- 행마다 핀이 반칸씩 어긋난 지그재그 구조, 총 10행
- 모든 핀에 Canvas `shadowBlur`를 이용한 은은한 glow 효과 적용

**물레 (빙글빙글 물레방아 맵 전용):** 십자 날개 4개짜리 물레 2개가 보드 중앙부에 배치된다. 각 물레는 시계방향으로 일정 속도로 회전하며 Matter.js의 키네마틱 바디로 구현한다. 구슬이 날개에 부딪히면 충돌 물리가 적용된다. 물레는 야광 초록색(#39FF14)으로 렌더링하며 강한 glow 효과를 적용한다. 물레가 있는 영역(보드 중앙 구역)에는 핀을 배치하지 않으며, 그 외 상하 영역은 기본맵과 동일한 지그재그 핀 배치를 유지한다.

**구슬:** `docs/images/pinball.png`를 베이스 이미지로 사용하고, Canvas의 오프스크린 합성(`globalCompositeOperation: multiply`)으로 플레이어별 색상을 적용한다. 30명까지 대비한 30개 색상 팔레트를 사전 정의한다. 총 1000개의 구슬이 지급되며, 인원수에 따라 균등 분배한다(예: 10명 참가 시 인당 100개). 구슬은 약 40ms 간격으로 대포에서 혼합 발사되며 전체 게임은 약 30~60초 소요된다.

**블랙홀:** `docs/images/hole.png` 이미지를 사용하며, 보드 하단부(보드 높이의 약 70~80% 지점) 중앙에 배치한다. 구멍은 Matter.js의 sensor body로 구현되며, 구슬이 들어가면 투명하게 사라지면서 해당 플레이어의 점수가 1점 올라간다.

**우승자 텍스트:** 우승이 확정되는 순간, 화면 우측 하단에 즉각적으로(fade-in 없이) 다음 형식의 텍스트가 표시된다.

```
Winner
플레이어A
```

- "Winner" 텍스트: 흰색, 30px 이상
- 플레이어 닉네임: 해당 플레이어의 구슬 색상과 동일한 색, 30px 이상
- fade-in 없이 즉시 표시

**컨페티 효과:** 우승 확정 순간, 화면 우측 하단에서 좌측 상단 방향으로 형형색색의 네모 박스 약 40개가 발사된다. 각 네모 박스는 포물선을 그리며 튀어오르고, 1~2초 간격으로 5회 반복 발사된다. 컨페티는 Canvas 또는 DOM 레이어에서 구현한다.

**설정 UI 복귀:** 우승이 확정되면(조기 확정·구슬 소진 모두) 화면 좌측 하단에 설정 화면의 입력 영역(textarea + 드롭다운 + 시작 버튼)이 약 1초에 걸쳐 CSS fade-in으로 나타난다. 게임(구슬 발사·물리)은 계속 진행된다.

## 블랙홀 모드
잔여 구슬이 200개 이하로 줄어들면 블랙홀 모드가 발동된다. 구멍이 점점 커지며, 회오리 애니메이션 대신 WebGL 쉐이더 기반 시각 효과(블랙홀이 빨아들이는 느낌의 왜곡/흡인 이펙트)가 적용된다. 화면 전체에 "BLACK HOLE MODE" 텍스트가 표시된다. Matter.js의 중력 방향이 구멍 쪽으로 전환되어 남은 모든 구슬이 빨려 들어가며 점수가 최종 합산된다.

## 피날레 연출 (잔여 구슬 10개)
잔여 구슬이 10개로 줄어들면 슬로우모션이 적용되고, 카메라가 hole 위치로 줌인되면서 게임 시간이 천천히 흐르는 연출이 재생된다.

## 우승 확정 조건
다음 두 조건 중 하나가 충족되면 우승이 결정되고 우측 하단에 우승자 배너가 표시된다.

1. **구슬 소진:** 총 1000개의 구슬이 모두 소진되었을 때, 구멍에 넣은 구슬 수가 가장 많은 플레이어가 우승한다.
2. **조기 확정:** 구슬이 모두 소진되기 전이라도, 현재 1위 플레이어의 점수가 압도적이어서 — 남은 구슬 전체가 2위 플레이어에게 몰아가더라도 1위를 역전할 수 없는 경우 — 즉시 우승이 확정된다.
   - 판정 공식: `1위 점수 > 2위 점수 + 잔여 구슬 수` 일 때 조기 확정
   - 조기 확정 후에도 구슬 발사는 계속되며, 설정 입력 영역이 fade-in으로 복귀

# 게임 흐름

설정 화면에서 닉네임과 맵을 선택하고 게임을 시작하면 입력창이 즉시 사라지고 대포에서 구슬 발사가 시작된다. 우측 상단 플레이어 목록은 유지되며 점수가 실시간 업데이트된다. 구슬이 핀(또는 물레)을 타고 낙하하며 구멍에 들어갈 때마다 점수가 집계된다. 매 구슬 소진마다 조기 확정 조건(1위 점수 > 2위 점수 + 잔여 구슬)을 검사한다. 잔여 구슬이 200개 이하가 되면 블랙홀 모드가 발동된다. 잔여 구슬 10개에서 슬로우모션 + 줌인 피날레 연출이 시작된다. 우승 확정(구슬 소진 또는 조기 확정) 시 화면 우측 하단에 우승자 텍스트가 즉시 표시되고, 컨페티가 우측 하단→좌측 상단 방향으로 5회 발사된다. 동시에 좌측 하단에 설정 입력 영역이 약 1초 fade-in으로 복귀한다.

# 핵심 기능

## P0 (출시 필수)
닉네임 콤마 파싱 및 유효성 검사, 맵 선택 드롭다운(설정 화면 입력 영역 내), 우측 상단 플레이어 목록(설정·게임 공통 HUD·플레이어 색상 텍스트), Matter.js 파칭코 보드(핀·중력·충돌), 고정 가상 해상도(1600×900) + CSS 스케일링, 대포 이미지 렌더링 및 발사 로직, 총 1000개 구슬 인원수 균등 분배(~40ms 간격 발사), 플레이어별 색상 구슬 발사(pinball.png 기반 Canvas 합성), 지그재그 격자 핀 배치(홀수행 6개·짝수행 7개·10행), 물레 키네마틱 바디(빙글빙글 물레방아 맵), 블랙홀 sensor 점수 집계, 블랙홀 모드(잔여 200개 이하·WebGL 쉐이더 흡인 이펙트·흡인력), 피날레 연출(잔여 10개·슬로우모션·줌인), 우승 확정 판정(구슬 소진 또는 조기 확정 공식), 우승자 텍스트(우측 하단 즉시 표시·플레이어 구슬 색상), 컨페티(우측 하단→좌측 상단·네모 박스 40개·5회 반복), 우승 확정 시 설정 입력 영역 fade-in 복귀(좌측 하단·약 1초)는 모두 필수 구현 항목이다.

# 비기능 요구사항

총 1000개 구슬이 동시에 시뮬레이션될 때 60fps를 유지해야 한다. 고정 가상 해상도(1600×900) 기준으로 물리 연산이 이루어지며, CSS transform으로 뷰포트에 맞게 스케일링되어 브라우저 크기와 무관하게 동일한 레이아웃을 보장한다. 화면은 데스크탑 기준으로 최적화하며 모바일 대응은 선택사항이다. 브라우저는 Chrome 최신 버전을 기준으로 하며, 게임 단계(playing / blackhole / finale)는 React 상태로 관리한다.

# 검증 기준

닉네임 2~30명 파싱이 정상 동작하고 우측 상단에 플레이어 목록(이름·구슬수·순서·색상)이 실시간 표시되어야 한다. 맵 선택 드롭다운이 정상 동작하고 선택한 맵에 따라 물레 유무가 달라져야 한다. 핀이 홀수행 6개·짝수행 7개·10행의 지그재그 격자로 올바르게 배치되어야 한다. 브라우저 크기를 변경해도 게임 레이아웃이 유지되어야 한다. 총 1000개 구슬이 약 40ms 간격으로 발사되어 30~60초 내 완료되어야 한다. 구멍에 들어간 구슬의 점수가 정확히 집계되어야 한다. 잔여 구슬 200개 이하일 때 블랙홀 모드와 WebGL 쉐이더 이펙트가 정상 발동되어야 한다. 잔여 구슬 10개일 때 슬로우모션 + 줌인 피날레 연출이 정상 재생되어야 한다. 조기 확정 공식(1위 점수 > 2위 점수 + 잔여 구슬)이 정확히 판정되어야 한다. 우승 확정 시 우측 하단에 위너 배너가 fade-in으로 표시되어야 한다. 조기 확정 시 게임이 계속 진행되며 좌측 하단에 설정 입력 영역이 fade-in으로 복귀되어야 한다.
