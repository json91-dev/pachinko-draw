# 개요
닉네임을 입력한 플레이어들이 각자의 색상 구슬을 파칭코 방식으로 떨어뜨려, 바닥의 구멍 1개에 가장 많이 넣은 플레이어가 당첨되는 웹 게임이다. 기술 스택은 Next.js와 Matter.js(물리 엔진)를 사용한다.

---

# 화면 구성

## 설정 화면
`docs/images/input.png`를 참조한 전체화면 입력 UI로 구성된다. 화면 하단에 "이름들을 입력하세요" 제목과 큰 textarea, "시작" 버튼이 위치한다. 콤마(,)로 구분하여 닉네임을 입력하면 상단 영역에 플레이어 카드가 실시간으로 자동 생성·제거된다(파싱된 이름 수만큼). 공백은 자동으로 제거하지 않으며 입력값 그대로 유지한다. 최소 2명에서 최대 30명까지 참여할 수 있다. 빈 닉네임과 중복 닉네임은 허용하지 않으며, 유효성 검사를 통과해야 "시작" 버튼이 활성화된다.

맵 선택 기능이 제공된다. 게임 시작 전 다음 두 맵 중 하나를 선택한다.
- **기본맵:** 핀이 불규칙하게 산포된 기본 파칭코 보드
- **빙글빙글 물레방아:** 십자 날개 4개짜리 물레 2개가 보드 중앙부에 배치되어 시계방향으로 일정 속도로 회전하는 맵. 물레가 있는 공간에는 핀이 없다.

게임 시작 버튼을 누르면 입력창과 카드 영역은 즉시 사라지고 게임 Canvas로 전환된다.

## 게임 화면
Canvas 기반으로 렌더링되는 파칭코 보드는 고정 핀(페그)과 좌우 벽, 상단 경계로 구성된다.

**대포:** 보드 상단 중앙에 `docs/images/cannon.png` 이미지로 대포를 렌더링하며, 구슬은 이 대포에서 발사된다. 대포는 좌우로 천천히 스윙(회전)하며, 현재 가리키는 각도 방향으로 구슬을 연속 발사한다.

**핀:** `docs/images/pin_128.png`(정사각형 핀) 이미지를 사용하며, `docs/images/paint.png` 스케치를 참조하여 불규칙하게 산포 배치한다 (완벽한 삼각형 격자 배열이 아님). 빙글빙글 물레방아 맵에서는 물레가 있는 영역에 핀을 배치하지 않는다.

**물레 (빙글빙글 물레방아 맵 전용):** 십자 날개 4개짜리 물레 2개가 보드 중앙부에 배치된다. 각 물레는 시계방향으로 일정 속도로 회전하며 Matter.js의 키네마틱 바디로 구현한다. 구슬이 날개에 부딪히면 충돌 물리가 적용된다.

**구슬:** `docs/images/pinball.png`를 베이스 이미지로 사용하고, Canvas의 오프스크린 합성(`globalCompositeOperation: multiply`)으로 플레이어별 색상을 적용한다. 30명까지 대비한 30개 색상 팔레트를 사전 정의한다. 총 1000개의 구슬이 지급되며, 인원수에 따라 균등 분배한다(예: 10명 참가 시 인당 100개). 구슬은 대포에서 혼합 발사된다.

**블랙홀:** `docs/images/hole.png` 이미지를 사용하며, 보드 하단부(보드 높이의 약 70~80% 지점) 중앙에 배치한다. 구멍은 Matter.js의 sensor body로 구현되며, 구슬이 들어가면 투명하게 사라지면서 해당 플레이어의 점수가 1점 올라간다. 화면 측면의 점수판(HUD)에는 플레이어별 현재 점수와 잔여 구슬 수가 실시간으로 표시된다.

**우승자 표시 (게임 중):** 우승이 확정되는 순간, 화면 우측 하단에 다음 형식의 위너 배너가 CSS fade-in 애니메이션으로 천천히 나타난다.

```
Winner
  플레이어A 🔵
```

- "Winner" 텍스트는 흰색, 30px 이상의 큰 글씨
- 플레이어 닉네임은 해당 플레이어의 구슬 색상으로 표시, 30px 이상
- 닉네임 옆에 해당 플레이어 색상의 구슬 이미지(pinball.png 합성) 표시

## 우승 확정 조건
다음 두 조건 중 하나가 충족되면 우승이 결정된다.

1. **구슬 소진:** 총 1000개의 구슬이 모두 소진되었을 때, 구멍에 넣은 구슬 수가 가장 많은 플레이어가 우승한다.
2. **조기 확정:** 구슬이 모두 소진되기 전이라도, 현재 1위 플레이어의 점수가 압도적이어서 — 남은 구슬 전체가 2위 플레이어에게 몰아가더라도 1위를 역전할 수 없는 경우 — 즉시 우승이 확정된다.
   - 판정 공식: `1위 점수 > 2위 점수 + 잔여 구슬 수` 일 때 조기 확정

## 블랙홀 모드
잔여 구슬이 200개 이하로 줄어들면 블랙홀 모드가 발동된다. 구멍이 점점 커지며, 회오리 애니메이션 대신 WebGL 쉐이더 기반 시각 효과(블랙홀이 빨아들이는 느낌의 왜곡/흡인 이펙트)가 적용된다. 화면 전체에 "BLACK HOLE MODE" 텍스트가 표시된다. Matter.js의 중력 방향이 구멍 쪽으로 전환되어 남은 모든 구슬이 빨려 들어가며 점수가 최종 합산된다.

## 피날레 연출 (잔여 구슬 10개)
잔여 구슬이 10개로 줄어들면 슬로우모션이 적용되고, 카메라가 hole 위치로 줌인되면서 게임 시간이 천천히 흐르는 연출이 재생된다. 마지막 구슬이 소진되면 결과 화면으로 전환된다.

## 결과 화면
우승이 확정되면 결과 화면으로 전환된다. 1위 닉네임이 화면 중앙에 초대형 폰트로 표시되며 컨페티 애니메이션이 재생된다. 하단에는 1위부터 최하위까지 전체 순위와 각 플레이어의 구슬 획득 수가 표시되며, 동점자는 공동 순위로 처리된다. 다시하기 버튼을 누르면 설정 화면으로 복귀한다.

# 게임 흐름

설정 화면에서 닉네임과 맵을 선택하고 게임을 시작하면 입력창이 즉시 사라지고 대포에서 구슬 발사가 시작된다. 구슬이 핀(또는 물레)을 타고 낙하하며 구멍에 들어갈 때마다 점수가 집계된다. 매 구슬 소진마다 조기 확정 조건(1위 점수 > 2위 점수 + 잔여 구슬)을 검사한다. 잔여 구슬이 200개 이하가 되면 블랙홀 모드가 발동된다. 잔여 구슬 10개에서 슬로우모션 + 줌인 피날레 연출이 시작된다. 우승 확정(구슬 소진 또는 조기 확정) 시 화면 우측 하단에 위너 배너가 fade-in으로 표시되며 결과 화면으로 전환된다. 다시하기를 누르면 설정 화면으로 돌아간다.

# 핵심 기능

## P0 (출시 필수)
닉네임 콤마 파싱 및 유효성 검사, 맵 선택(기본맵 / 빙글빙글 물레방아), 실시간 점수판 자동 생성, Matter.js 파칭코 보드(핀·중력·충돌), 대포 이미지 렌더링 및 발사 로직, 총 1000개 구슬 인원수 균등 분배 및 플레이어별 색상 구슬 발사(pinball.png 기반 Canvas 합성), 물레 키네마틱 바디(빙글빙글 물레방아 맵), 블랙홀 sensor 점수 집계, 블랙홀 모드(잔여 200개 이하·WebGL 쉐이더 흡인 이펙트·흡인력), 피날레 연출(잔여 10개·슬로우모션·줌인), 우승 확정 판정(구슬 소진 또는 조기 확정 공식), 우승자 배너 (우측 하단 fade-in·플레이어 색상·구슬 이미지), 결과 화면(1위 강조·순위표), 점수판 HUD 실시간 업데이트는 모두 필수 구현 항목이다.

## P1 (권장)
구슬 발사 시 순차 딜레이를 적용한 애니메이션을 구현한다. 효과음을 추가한다.

# 비기능 요구사항

총 1000개 구슬이 동시에 시뮬레이션될 때 60fps를 유지해야 한다. 화면은 데스크탑 기준으로 최적화하며 모바일 대응은 선택사항이다. 브라우저는 Chrome 최신 버전을 기준으로 하며, 게임 단계(setup / playing / blackhole / finale / result)는 React 상태로 관리한다.

---

# 파일 구조

app 하위에 설정 화면을 담당하는 page.tsx가 위치하고, game 폴더 안에 게임 화면(page.tsx)과 PachinkoBoard, Windmill, BlackHole, Scoreboard, WinnerBanner 컴포넌트가 들어간다. 결과 화면은 game/result/page.tsx에 위치하며 WinnerDisplay와 RankingTable 컴포넌트로 구성된다. 구슬 색상 팔레트와 Canvas 합성 유틸리티는 lib/ballTint.ts에 분리하고, WebGL 쉐이더 유틸리티는 lib/blackholeShader.ts에 분리한다. 우승 확정 판정 로직은 lib/winnerCheck.ts에 분리한다.

# 검증 기준

닉네임 2~30명 파싱이 정상 동작하고 실시간으로 점수판이 생성되어야 한다. 맵 선택이 정상 동작하고 선택한 맵에 따라 물레 유무가 달라져야 한다. 총 1000개 구슬이 인원수에 따라 균등 분배되어야 한다. 대포에서 구슬이 자연스럽게 발사되어야 하고, 핀(또는 물레)에 튕기며 자연스럽게 낙하해야 한다. 구멍에 들어간 구슬의 점수가 정확히 집계되어야 한다. 잔여 구슬 200개 이하일 때 블랙홀 모드와 WebGL 쉐이더 이펙트가 정상 발동되어야 한다. 잔여 구슬 10개일 때 슬로우모션 + 줌인 피날레 연출이 정상 재생되어야 한다. 조기 확정 공식(1위 점수 > 2위 점수 + 잔여 구슬)이 정확히 판정되어야 한다. 우승 확정 시 우측 하단에 위너 배너가 fade-in으로 정상 표시되고 닉네임이 해당 플레이어 색상으로 렌더링되어야 한다. 결과 화면에서 1위와 전체 순위가 정확히 표시되고 다시하기 버튼으로 설정 화면 복귀가 정상 동작해야 한다.
